How to make a blog like this one

Posted: 2021-01-02   |   Categories: technology   |   Tags: github, hugo

This post is about how to make a blog for your own that resembles this one. In this post, I will make a step-by-step guide about how I made this possible so that those who want to do something similar can do so.

My main target for this is will be linux and mac users. There are no prerequisites, so that it is understood by almost anyone who has some familiarity with the command line. I will explain some concepts very briefly as we encounter it, for completeness.

Most of all, the purpose of this post is to help SS (a friend) come up with a blog. SS is a mac user and I really enjoy reading/listening to the stuff SS creates so I think it would look good on a blog. I will read it, certainly.

How it works

How does a website load into your browser? When somebody opens a url on the browser, the browser goes to a DNS (Domain Name Setup) lookup service to find the IP address from which some html, css, javascript, etc. code can be downloaded for the browser to process.

Websites could be either static or dynamic. A static website does not interact with the server in real-time to change the content of the website. Such websites are rare. Dynamic websites such as facebook are constantly interacting with their servers to load content (and to store your data, but anyway…).

This website is mostly static, except for some parts like the comments section below, and the frame containing my Mastodon toots on the main page. git, github and all that

git is a program that was written by Linus Torvalds, one of the creators of linux. It allows one to keep track of code as it develops and allows it to revert back to a previous state when something goes wrong. It also facilitates branching of code development for the collaboration by multiple developers.

Initially, git was used for collaboration on local networks. With the advent of the internet, websites like github and gitlab started to use this protocol for “pushing” the code on the cloud. Eventually, github came up with an interesting feature: if the code being uploaded is that of a site (meaning, it contains html, css, javascript, etc.), github will display it as a site.

So how do you go about this? If you don’t have one, create an account on github. Suppose your username is username, then create a repository called username.github.io. Then open that repository and click on the button that says Code.

Then, do the following in a terminal (of course, can choose some other folder than ~/blogsite if you want)

mkdir blogsite
cd blogsite
